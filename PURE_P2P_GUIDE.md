# 🌐 纯 P2P 去中心化聊天（无需任何服务器）

## 🎯 核心特性

- ✅ **无需任何服务器** - 真正的 P2P
- ✅ **无需公网 IP** - 同一局域网即可
- ✅ **无需端口转发** - 直接连接
- ✅ **自动保存记录** - 本地保存聊天
- ✅ **完全去中心化** - 没有任何中心节点

---

## 📋 三种使用场景

### 场景 1: 同一局域网（最简单）⭐

**适用**: 你们在同一个 WiFi/路由器下

**A 启动监听**:
```bash
node pure-p2p.js listen 8091
# 输出：我的编码：OCLAW-AAAA-BBBB-CCCC
```

**B 主动连接**:
```bash
node pure-p2p.js connect 192.168.1.100 8091 OCLAW-AAAA-BBBB-CCCC
```

**成功率**: ✅ 100%（同一局域网）

---

### 场景 2: 一方有公网 IP

**适用**: 至少一方有公网 IP（如公司网络）

**A（有公网 IP）启动监听**:
```bash
node pure-p2p.js listen 8091
# 告诉 B 自己的公网 IP 和编码
```

**B 主动连接**:
```bash
node pure-p2p.js connect A 的公网 IP 8091 OCLAW-AAAA-BBBB-CCCC
```

**成功率**: ✅ 100%

---

### 场景 3: 双方都无公网 IP

**适用**: 双方都在不同内网

**很遗憾**: ❌ 无法直接通信

**原因**:
- 没有公网 IP，对方无法找到你
- 没有服务器中转，无法建立连接

**解决方案**:
1. 使用信令服务器方案（需要一台云服务器）
2. 使用第三方免费中继服务
3. 其中一方使用手机热点（获取公网 IP）

## 🚀 快速开始

### 步骤 1: 生成你的连接信息

```bash
node pure-p2p.js generate
```

**输出**:
```
========================================
📝 你的连接信息
========================================
编码：OCLAW-AAAA-BBBB-CCCC
端口：8091 (默认)
========================================
```

### 步骤 2: 启动监听（等待对方连接）

```bash
node pure-p2p.js listen 8091
```

**输出**:
```
========================================
✅ P2P 监听服务已启动
========================================
📍 我的编码：OCLAW-AAAA-BBBB-CCCC
📍 监听端口：8091
========================================
```

### 步骤 3: 告诉对方你的信息

```
我的编码：OCLAW-AAAA-BBBB-CCCC
我的 IP: 192.168.1.100（局域网 IP）
我的端口：8091
```

### 步骤 4: 对方连接你

```bash
node pure-p2p.js connect 192.168.1.100 8091 OCLAW-AAAA-BBBB-CCCC
```

### 步骤 5: 开始聊天

```
你：你好啊！
对方：你好！收到你的消息了
```

## 🔍 获取局域网 IP

### Windows
```cmd
ipconfig
# 找到 IPv4 地址：192.168.x.x
```

### macOS/Linux
```bash
ifconfig | grep "inet "
# 找到 inet 192.168.x.x
```

```bash

hostname -I

# macOS
```

## ❓ 常见问题

### Q1: 连接超时

**症状**: `⏰ 连接超时`

**原因**:
- IP 地址错误
- 对方未启动监听
- 防火墙阻止
- 不在同一网络

**解决方案**:
1. 确认 IP 地址正确
2. 确认对方已启动 `listen`
3. 关闭防火墙测试
4. 确认在同一局域网

### Q2: 找不到对方

**症状**: 连接后没有响应

**解决方案**:
1. 确认对方防火墙允许 UDP 8091
2. 确认对方程序正在运行
3. 尝试 ping 测试：`ping 192.168.1.100`

### Q3: 双方都无公网 IP

**症状**: 无法建立连接

**解决方案**:
1. 使用信令服务器方案
2. 其中一方使用手机热点
3. 使用第三方中继服务
