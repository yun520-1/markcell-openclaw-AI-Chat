# 🌐 OpenClaw 外网服务配置清单

## 📋 配置步骤

### 1️⃣ 启动外网服务

```bash
cd ~/.qclaw/workspace/skills/markcell-openclaw-ai-chat
node wan-chat.js
```

**预期输出**:
```

========================================

📍 局域网 IP: 192.168.x.x
📍 公网 IP: 49.87.224.177
📍 端口：1238
📍 外网地址：http://49.87.224.177:1238

✅ 外网对话服务已启动，等待连接...
```

---

### 2️⃣ 确认实际端口

查看启动输出中的端口号：
- 如果是 **1238**，继续使用
- 如果是其他数字，请使用实际端口

---

### 3️⃣ 配置防火墙（云服务器必做）

#### 阿里云 ECS
1. 登录阿里云控制台
2. 进入 **云服务器 ECS**
3. 找到你的实例
4. 点击 **安全组**
5. 点击 **配置规则**
6. 添加 **入站规则**:
   - 授权策略：允许
   - 协议类型：TCP
   - 端口范围：**1238/1238**
   - 授权对象：0.0.0.0/0
   - 优先级：1

#### 腾讯云 CVM
1. 登录腾讯云控制台
2. 进入 **云服务器**
3. 找到你的实例
4. 点击 **安全组**
5. 点击 **修改规则**
6. 添加 **入站规则**:
   - 类型：自定义
   - 协议端口：TCP:1238
   - 来源：0.0.0.0/0
   - 策略：允许

#### 华为云 ECS
1. 登录华为云控制台
2. 进入 **弹性云服务器**
3. 找到你的实例
4. 点击 **安全组**
5. 点击 **配置规则**
6. 添加 **入站规则**:
   - 协议/端口：TCP:1238
   - 源地址：0.0.0.0/0
   - 策略：允许

---

## 🚨 常见问题

### 问题 1: 端口被占用

**症状**: `Error: listen EADDRINUSE: address already in use 0.0.0.0:1238`

**解决方案**:
```bash
# 找到占用端口的进程
lsof -i :1238

# 杀死进程
kill -9 [PID]

# 或者修改端口
# 编辑 wan-chat.js，修改 port: 1238 为 port: 1239
```

### 问题 2: 无法获取公网 IP

**症状**: `curl ifconfig.me` 无输出或报错

**解决方案**:
```bash
# 尝试其他服务
curl api.ipify.org
curl ifconfig.co
```

### 问题 3: 外网无法访问

**症状**: 对方测试连接超时

**解决方案**:
1. 确认安全组已配置
2. 确认防火墙已开放端口
3. 确认公网 IP 正确
4. 测试本地访问是否正常
